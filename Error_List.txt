  [ERROR-01] 세로 영상에서 스케일 버그 ★ 수정 가능
────────────────────────────────────────────────────────────────
파일: src/video_panel.py — _apply_overlay()

문제:
  현재 코드: _sc = min(1.0, 640 / max(w_px, 1))
  → 너비(w_px) 기준으로만 스케일 계산
  → 세로 영상(1080x1920)에서 640/1080 = 0.593
  → 추론 해상도: 640x1137 (높이가 1137px로 비정상적으로 큰 이미지)

올바른 방법:
  _sc = min(1.0, 640 / max(w_px, h_px, 1))
  → 최대 변(높이 1920) 기준으로 계산 → 0.333
  → 추론 해상도: 360x640 (정상적인 비율)

영향:
  - 가로 영상(1920x1080): 현재 코드도 정상 동작 (너비가 최대 변)
  - 세로 영상(1080x1920): 현재 코드가 640x1137 생성 → 비정상 비율로
    감지 모델에 입력 → 정확도 저하 가능
  - 동일 버그가 _save_video_frames() 애니화 감지 코드에도 존재

수정:
  _sc = min(1.0, 640 / max(w_px, h_px, 1))  ← 두 곳 수정 완료
