## 작업: puppet_pin.py 메시 방식 재설계 (Live2D 메시 구조 참조)

### 배경
- 기존 2-세그먼트 warpAffine 방식: 팔 이미지가 크면 화면 전체를 오염
- 근본 원인: "메시 밖 픽셀은 렌더하지 않는다"는 경계가 없음

### 구현 내용 (`src/puppet_pin.py` 전면 재작성)

#### 새 구조
- `SegmentCache`: `img_orig` + `pins` + `mesh_verts(N,2)` + `mesh_tris(M,3)` + `vert_seg(N,)`
- Live2D `vertex_pos` ↔ `mesh_verts`, `vertex_indices` ↔ `mesh_tris`

#### 신규 함수
1. `_build_arm_mesh(pins, img_h, img_w, n_segs=8)` — Strip Mesh 생성
   - 총 vertex: 4N+2 = 34 (N=8), 총 triangle: 4N = 32
   - elbow vertex 공유로 seam 없음
2. `_deform_mesh(...)` — 이미지 공간 → 비디오 공간 vertex 변환 (행렬 곱 벡터화)
3. `_mesh_to_remap(...)` — 삼각형별 역방향 affine → map_x/map_y
4. `_pts_in_triangle(pts, tri)` — Vectorized barycentric 테스트

#### 핵심 효과
- `cv2.remap` + `BORDER_CONSTANT` + map=-1: 메시 밖 픽셀 자동 alpha=0
- 더 이상 배경을 덮는 문제 없음
