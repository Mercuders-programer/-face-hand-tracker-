### 2. 영상 분석 — AnimeGAN ONNX 애니화 구현 (STEP 2 우선)

#### 신규 파일: `src/anime_converter.py`
- `build_person_mask(w, h, pose_res, face_res, hand_res, dilate_px=50, blur_k=31)`
  - 포즈 33점 + 얼굴 bbox + 손 hull → Convex Hull → dilate → GaussianBlur 경계
  - 감지 없으면 전체 마스크 반환 (폴백)
- `apply_opencv_anime(frame, scale=512)`
  - bilateral × 7 + adaptiveThreshold 엣지 + bitwise_and 합성
  - 내부 scale 축소 처리 후 원본 해상도 복원
- `class AnimeGANConverter`
  - `load(model_path)`: ONNX 모델 로드, CUDAExecutionProvider 우선
  - `convert(frame, scale=512)`: 32배수 리사이즈 → ONNX 추론 → BGR 복원
  - `loaded` property
- `apply_anime_to_person(frame, pose_res, face_res, hand_res, style, bg_mode, converter)`
  - 마스크 생성 → 스타일 변환 → 배경 합성 (원본/블러/단색)
  - AnimeGAN 실패 시 OpenCV 자동 폴백

### 3. 얼굴 감지 실패 상세 진단 (diagnose_face.py)

#### 신규 파일: `diagnose_face.py`
- conf(0.5/0.3/0.1/0.05) × 해상도(원본/640/360) 12조합 전수 테스트
- PoseLandmarker nose 랜드마크로 얼굴 방향 역산
- OpenCV Haar Cascade (정면+측면) 참조 테스트
- 첫 프레임 이미지 저장 (TEST_DATA/diagnose_frame1.jpg)

#### 핵심 발견
- **conf=0.05 + 202×360 해상도**에서 얼굴 1명 감지 성공
  - yaw=-68.6°, pitch=+21.4° → 왼쪽으로 거의 측면(68.6°)
- 현재 코드 기본 confidence(0.5)는 이 각도에서 score가 임계값 미달
- Haar Cascade는 원본/640px에서 감지 성공 → MediaPipe보다 관용적
- 포즈 기반 yaw 추정은 "몸=정면, 얼굴=측면" 자세에서 오판

#### 신규 파일: `Error_analyze.txt`
- SECTION 1: FaceLandmarker 전수 테스트 표
- SECTION 2: BlazeFace (모델 없어서 스킵)
- SECTION 3: Pose nose 랜드마크 분석 + yaw 추정 오판 원인
- SECTION 4: Haar Cascade 결과
- SECTION 5: 원인 4가지 상세 분석
- SECTION 6: 해결방안 A~E + 권장(conf=0.05 + 360px 즉시 적용 가능)
